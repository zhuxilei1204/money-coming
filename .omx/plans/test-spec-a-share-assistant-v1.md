# Test Spec：A股交易决策支持助手 V1

## 1. 测试目标
验证 V1 规划是否能支持一个安全、可回溯、非实时、非自动交易的信息聚合与方向性倾向辅助闭环。

## 2. 测试范围
### 在范围内
- 有限观察池/持仓输入
- 免费源/离线样例驱动的数据处理与摘要
- 证据链、时间戳、反证、不确定性、免责声明、人工确认、审计字段
- 持仓与非持仓两类输出差异
- 降级/拒绝输出路径

### 不在范围内
- 自动交易/券商执行
- 全市场扫描
- 付费源
- 高频实时监控

## 3. 测试分层
### A. 规格一致性测试
- PRD 是否保留全部硬约束
- 所有 lane 是否有明确输入/输出/交接
- 用户确认点是否完整列出

### B. 数据契约测试
- 单条证据是否有来源、时间戳、原文/摘要、实体关联、批次/审计字段
- 是否包含最小契约字段：`evidence_id/source_id/source_type/captured_at/event_time/entity_scope/content_hash/provenance_level/batch_id/audit_id`
- 重复事件是否可被识别或标记
- 缺失字段是否被显式标注
- 过期或时效不明证据是否被降权或隔离

### C. 摘要输出测试
- 摘要是否包含事实、影响方向、证据强弱、不确定性
- 是否可追溯到原始证据

### D. 倾向输出测试
- 输出是否为非二元分数而非命令式建议
- 是否同时包含正向因素、负向因素、反证、置信度、免责声明、人工确认
- 缺证据时是否降级为“不形成倾向”
- 无公司级/个股级证据时，是否禁止高置信打分
- `summary_only/no_score/low_confidence_score/normal_score` 状态是否符合触发规则

### E. 安全审查测试
- 是否绝不出现自动下单/自动执行链路
- 是否拒绝无免责声明或无时间戳输出
- 是否拒绝只基于主题噪声、无个股证据支撑的高置信倾向
- 来源许可不合规时，是否进入 `reject_output`
- 持仓/非持仓输出路径是否正确区分字段与提示语
- 免责声明/人工确认/审计字段缺失时，是否进入 `reject_output`

### F. 端到端样例测试
- 样例 1：已持仓标的 + 公告/新闻/行业背景 -> 输出持仓视角摘要
- 样例 2：观察名单标的 + 多源信息 -> 输出观察视角倾向
- 样例 3：主题热度很高但个股证据弱 -> 降级/谨慎输出

## 4. 最小夹具设计
### Fixtures
1. **macro_news_fixture**：指数/政策/宏观新闻样例
2. **company_announcement_fixture**：上市公司公告样例
3. **sector_theme_fixture**：板块/题材资讯样例
4. **position_input_fixture**：持仓输入样例
5. **conflict_evidence_fixture**：正反证据冲突样例
6. **missing_fields_fixture**：来源缺时间戳/字段缺失样例
7. **duplicate_event_fixture**：同事件多源重复样例

## 5. 验收案例矩阵
### Case 1：正常持仓分析
- 输入：持仓样例 + 公告 + 宏观背景
- 期望：输出包含持仓视角、分数、正反因素、不确定性、免责声明、审计字段

### Case 2：正常观察名单分析
- 输入：观察标的 + 行业/主题背景 + 新闻
- 期望：输出非持仓视角倾向，且主题只做辅助

### Case 3：证据不足
- 输入：只有零散主题热度，无可靠个股证据
- 期望：输出“信息不足/不形成倾向”

### Case 4：证据冲突
- 输入：利好公告 + 明显利空新闻
- 期望：输出必须同时展示反证与不确定性，分数置信度下降

### Case 5：字段缺失
- 输入：缺来源时间戳的数据
- 期望：该数据被标记低可信或不进入主结论

### Case 6：重复事件
- 输入：同一公告被多个源转载
- 期望：可识别重复，不重复放大利好/利空

### Case 7：时效过期
- 输入：过旧新闻与较新公告混合
- 期望：旧证据被降权，不能单独支撑正常分数；若只剩过期证据，则降到 `no_score` 或 `reject_output`

### Case 8：来源许可不合规
- 输入：来源内容可访问但许可不明或不允许该使用方式
- 期望：该来源被阻断或仅作为待确认候选，不进入主结论

### Case 9：仅有主题噪声
- 输入：题材热度很高，但无公司级证据
- 期望：最高只能 `no_score`，且必须标注主题映射推断

### Case 10：持仓/非持仓路径差异
- 输入：同一标的分别以持仓与观察名单身份分析
- 期望：持仓路径多出仓位/成本相关提示，非持仓路径不出现持仓假设

## 6. 通过标准
- 每条最终输出都满足最小字段集合。
- 每条倾向输出都可追溯到至少一条有效证据。
- 所有高风险场景都有降级、拒绝或显式提示机制。
- 任一测试都不依赖实时/付费数据源。

## 7. 失败即阻塞项
- 出现命令式买卖建议。
- 无法回溯证据来源与时间戳。
- 主题噪声可直接驱动高置信倾向。
- 缺免责声明/人工确认语句。
- 存在自动交易接口设计。

# Test Spec: A股交易辅助帮手 V1 角色与功能细化

## 1. 验证目标
验证本轮规划是否把角色职责、V1 功能要求、安全护栏、确认门和后续执行边界说清楚。此 test spec 验证规划质量，不要求当前已创建生产代码、`PROJECT.md` 或真实数据接入。

## 2. 规划文档验收
- PRD 中每个角色都有：关注点、输入、任务、输出、交接。
- PRD 明确区分“本轮规划产物”与“后续执行产物”。
- PRD 定义 `PROJECT.md` 结构/维护规则和 `AGENTS.md` 后续约束要求，但不把实际创建/修改作为本轮完成条件。

## 3. V1 功能清单验收
- 明确三类目标对象：持仓、自选、指定板块/主题。
- 信息聚合最小字段包含：标题、来源、发布时间、记录时间、关联标的/板块、事件类型、摘要、证据链接/引用、数据质量标记。
- 单次分析最小字段包含：事实摘要、风险/机会、反方因素、不确定性、置信度、适用前提、失效条件、评分式倾向、免责声明、人工确认提示。
- 审计记录最小字段包含：输入快照、来源、时间戳、模板版本、评分版本、输出、人工确认状态、agent/运行人、错误或缺口。

## 4. 安全验收
- 规划禁止自动交易、下单、券商订单 API、全市场覆盖、付费源 V1、实时高频。
- 规划禁止命令式/二元化买卖话术。
- 规划要求评分式倾向必须有证据、不确定性、适用前提、失效条件、免责声明、审计记录和人工确认。
- 已持仓与未持仓的分析上下文必须不同。

## 5. 用户确认 Gate 验收
PRD 必须声明以下事项确认前不得进入实现：技术栈与目录结构、具体数据源接入、评分公式与模板、首版交互形态、任何付费源或更高频更新。

## 6. 后续测试策略
- Unit: 字段校验、解析/去重、评分输入校验、安全措辞检查。
- Integration: 离线 fixture 到摘要/分析样例的完整链路。
- Regression: 固定 fixture 输出稳定；护栏规则不被破坏。
- Manual review: 数据源候选、评分模板、交互形态需用户确认。

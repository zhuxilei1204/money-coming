# 测试 / 验收工程师

## 使命
把角色职责、数据源、分析输出和安全护栏转化为可验证的测试、fixture、验收清单和回归检查。

## 关注点
- 离线可测，不依赖付费或实时源。
- 字段完整性、护栏完整性、回归稳定性。
- 持仓/未持仓样例差异。
- 完成声明必须有证据。

## 输入
- PRD 与 Test Spec
- 角色文档
- fixture 规范
- 分析样例与安全 checklist

## 要做什么
- 设计 fixture schema 和 mock 源。
- 定义字段校验、安全措辞校验、审计字段校验。
- 设计从离线输入到摘要/分析输出的集成测试。
- 维护验收清单和测试结果记录。

## 怎么做
1. 先用离线样例覆盖持仓、自选、板块三类对象。
2. 对每个分析输出检查最小字段：证据、不确定性、适用前提、失效条件、评分、免责声明、人工确认。
3. 加入负例：命令式买卖建议、缺失来源、缺失审计字段、误用持仓上下文。
4. 每次实现后记录测试命令、结果、未测试风险。

## 做到什么样
- 没有测试证据不得宣称完成。
- 安全边界被回归测试锁定。
- fixture 可重复运行，输出差异可解释。

## 不能做什么
- 不能依赖真实付费源或实时网络作为唯一测试路径。
- 不能忽略安全负例。
- 不能只测 happy path。
